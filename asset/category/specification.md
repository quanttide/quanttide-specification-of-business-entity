# 工程标准资产

## 概述

工程标准资产是规范化工程实践的文档资产，用于标准化开发流程、文档格式、工具使用等方面。

## 用途

- 统一团队协作规范
- 提高文档和代码质量
- 降低沟通和理解成本
- 便于新人快速上手

## 创建标准

### 识别领域

需要创建标准的场景：
- 重复出现的问题
- 团队协作混乱
- 文档质量参差不齐
- 缺乏明确的工作流程

### 编写流程

1. **调研**：了解现有实践和社区标准
2. **编写**：参考文档格式规范撰写
3. **审查**：团队评审可行性
4. **发布**：创建版本并推广

### 标准结构

工程标准文档应包含：
- 概述：标准的用途和适用范围
- 规范内容：具体的标准要求
- 参考文档：引用的外部资料

## 维护标准

### 更新触发

以下情况需要更新标准：
- 实践中发现问题
- 工具或流程变更
- 团队反馈建议
- 外部标准更新

### 更新流程

1. 修改标准内容
2. 更新CHANGELOG
3. 更新版本号
4. 发布新版本
