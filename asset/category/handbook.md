# 手册资产

## 概述

手册资产是面向人类的操作经验文档，记录如何完成特定任务。

与工程标准的区别：
- 工程标准：定义规范和要求
- 手册资产：记录操作方法和实践经验

## 写作原则

- **只说怎么做**：不解释为什么
- **引用而非重复**：直接给出规范链接
- **版本化链接**：一句话说明版本更新方法
- **删除冗余**：不写"相关文档"、"版本更新"详细说明、原理解释

## 内容结构

1. 规范来源：直接链接 + 版本更新方法
2. 操作方法：清晰的步骤
3. 注意事项：必要的关键提醒

## 示例对比

**冗余写法**：
```markdown
## 链接不可变性
使用版本化的链接，而不是main分支链接。
- ✅ 正确：blob/v0.1.0/...
- ❌ 错误：blob/main/...
版本化链接确保规范内容不可变...
```

**极简写法**：
```markdown
## 规范来源
版本发布操作遵循量潮科技工程标准的版本发布标准
https://github.com/quanttide/quanttide-specification-of-business-entity/blob/v0.1.0/devops/release.md
当specification发布新版本时，更新本文档中的版本号和链接。
```

## 维护

更新时机：发现更好方法、规范版本更新、收到反馈

更新原则：保持简洁、删除过时、合并重复、引用而非重复
